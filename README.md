# RawResourceBase — 原始资料知识库

> 一个基于 Obsidian + Git 的个人投资知识库，聚焦 A 股交易、期货、产业研究等领域。
> 核心能力：将 B 站直播的 ASR 转录文稿，通过 AI 方法论加工为结构化的可检索知识。

---

## 项目简介

本仓库是一个 **Obsidian 知识库**，用于收集、整理和沉淀中文投资交易领域的原始资料。内容来源包括 B 站投资类 UP 主的直播转录、深度访谈、交易方法论文章、产业研报等。

项目的一大特色是建立了一套 **「直播视频文稿 AI 总结方法论」**，能够系统化地将海量、低信噪比的 ASR 转录文本，转化为结构清晰、可检索、可追踪知识演进的 Markdown 文档。

## 内容概览

### 🎙️ 直播转录与 AI 总结

收录三位 B 站投资类 UP 主的直播转录及 AI 总结：

| UP 主 | 内容特点 | 转录文件数 | 时间跨度 |
|--------|---------|-----------|---------|
| **无名游资** | 超短线交易、情绪周期、板块轮动、期货模拟 | 147 篇 | 2026.05 - 2026.07 |
| **做多中国的好人豆子** | 价值投资、猪周期/牧原、期货视角、宏观分析 | 102 篇 | 2026.04 - 2026.07 |
| **黄阳** | 学习分享与投资交流 | 162 篇 | 2026.04 - 2026.07 |

每篇转录文稿配有 AI 生成的结构化总结，包含：
- **Frontmatter 元数据**：视频 BVID、上传时间、时长估算、是否含金融内容
- **核心立场**：UP 主本场主要论点及重要度评级
- **金融与交易知识点**：按重要性排序的论证链（含⭐评级）
- **个人成长/自评**：人格色彩与情绪状态记录

### 📌 核心方法论

[`直播视频文稿AI总结方法论.md`](./直播视频文稿AI总结方法论.md) 是本项目的核心文档，定义了从 ASR 文稿到结构化知识的完整处理流程：

1. **前置判断**：根据文件大小和标题关键词快速分档（A/B/C 类），决定输出粒度
2. **去噪解析**：处理 ASR 字间空格格式，识别"伪内容"与"真实内容段"
3. **知识提取**：提取框架性观点、反直觉洞察、量化数据，剔除游戏/闲聊噪音
4. **结构化输出**：统一的 Frontmatter + "倒金字塔"正文结构
5. **场景化策略**：针对盯盘场、连麦答疑、专题讲座、游戏娱乐场分别制定策略
6. **跨文档管理**：跟踪知识演进、建立核心观点索引
7. **质量自检**：7 项检查清单确保产出质量

## Frontmatter 规范

本仓库所有 Markdown 文件统一使用 YAML Frontmatter，分为通用字段和转录专用字段：

```yaml
---
# ── 通用字段（所有文件必填）──
title: 文章标题
source_type: article              # article=外部转载 | transcript=直播转录
source_platform: 雪球              # 来源平台
source_url: https://...
author: 作者
publish_date: YYYY-MM-DD

# ── 直播转录专用（仅 source_type=transcript 时填写）──
source_video: BV1xxx              # B 站视频 BVID
duration_estimate: ~N 分钟
generated_at: YYYY-MM-DD          # AI 总结生成日期
has_finance_content: true/false
---
```

Frontmatter 之后正文以一级标题 `# 文章标题` 开头。

## 快速开始

### 作为 Obsidian 知识库使用

1. 安装 [Obsidian](https://obsidian.md/)
2. 选择「Open folder as vault」，打开本仓库目录
3. Obsidian 会自动加载 `.obsidian/` 中的配置和插件
4. 如需 Git 同步，安装 `obsidian-git` 社区插件

### 使用 AI 方法论处理新转录文稿

1. 将 ASR 转录的 `.txt` 文件放入对应 UP 主的 `直播转录/` 目录
2. 阅读 [`直播视频文稿AI总结方法论.md`](./直播视频文稿AI总结方法论.md) 了解处理流程
3. 按方法论生成结构化 `.md` 总结，放入对应的 `转录AI总结/` 目录
4. AI Agent 操作规范请参考 [`AGENTS.md`](./AGENTS.md)

### 从 Git 克隆

```bash
git clone https://github.com/StormPhoenix/RawResourceBase.git
```

## 文件命名约定

直播转录文件遵循统一命名格式：

```
YY_MM_DD-HH-MM-SS_标题截断_BVID.txt
```

示例：`26_05_22-03-28-54_【个人观点】你们明天可能会吃肉，而不是割肉。_BV1PuLY6gETp.txt`

AI 总结文件与源文件同名，扩展名为 `.md`。

外部转载文章（知乎/雪球等）使用日期前缀命名格式：

```
YYYY-MM-DD_文章标题.md
```

示例：`2024-08-04_优势投资.md`
