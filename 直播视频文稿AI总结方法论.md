# 直播视频文稿 AI 总结方法论

> **创作背景**：连续处理了 67+ 篇 B 站投资类直播的 ASR 转录文稿（每篇 5KB~186KB，累计 2.8MB+），
> 从中提炼出可供其他 LLM 复用的系统化总结方法论。
> 
> **输入特点**：ASR 输出为**字间空格格式**（如"我 今 天 讲 讲 市 场"），单行大文件，无标点、无分段。
> **输出目标**：结构化 Markdown 知识文档，含 frontmatter、核心立场、知识点归类、元标注。

---

## 一、前置判断：是否需要激活此流程

接触任何 ASR 文稿前，先判断：

### 1.1 快速评估内容密度（5 秒扫描法）

根据**文件大小 + 标题关键词**快速分档：

| 文件大小 | 典型内容 | 处理深度 |
|----------|----------|---------|
| **< 8 KB** | 开播测试、秒关、静默 | A 类：1-2 行说明，标记 `has_finance_content: false` |
| **8-40 KB** | 短答疑、部分闲聊夹杂金融 | B 类：中粒度，只摘 2-5 个核心知识点 |
| **40-80 KB** | 正常连麦/答疑/讲座 | C 类：深度总结，6-12 个知识点 |
| **> 80 KB** | 长场多人连麦/专题 | C 类++：可能需要分主题结构 |

**标题关键词预判**（中文场景）：
- **高概率有干货**：答疑、讲讲、市场、交易、期货、价值投资、预期、讲解
- **中概率有干货**：答疑、山崩、谷底、养/猪、牧原、山还没崩
- **低概率有干货**：MC、炉石、PVZ、原神、打游戏、启动、等待

### 1.2 决定输出粒度

| 级别 | 产出 | MD 大小 | 适用场景 |
|------|------|---------|---------|
| **A 类** | frontmatter + 1-2 行说明 | ~300-500 字节 | 纯游戏、极短场（<8KB） |
| **B 类** | frontmatter + 少量知识点 | ~1-3 KB | 游戏+闲聊混合场 |
| **C 类** | 完整结构化 MD | ~5-15 KB | 金融/交易主题场 |
| **C 类++** | 分主题 + 附洞察提炼 | ~10-15 KB | 长场（>80KB）多主题场 |

## 二、核心处理流程

### 2.1 阅读与解析（去噪阶段）

ASR 字间空格格式严重降低可读性。在 LLM 上下文中 **先做一次心理去空格**（即隐形剥离空格后理解语义）。具体做法：

1. **快速扫描开头 10%**：判断本场基调（答疑/游戏/闲聊/混合）
2. **识别"伪内容"标记**：
   - 大量重复无意义词（"嗯 嗯 嗯" "啊 啊 啊" "哼"）→ 等待加载/思考过渡
   - 游戏音效描述 → 纯游戏，可跳过
   - 弹幕互动（"感谢xxx的舰长" "家人们"）→ 社交填充，无金融价值
3. **锁定"真实内容段"**：当出现以下关键词时，进入精读模式：
   - 逻辑论述类：因为、所以、本质、核心、我认为、我的看法
   - 交易框架类：预期差、赚钱效应、确定性、模式、纪律
   - 市场判断类：黄金、牧原、生猪、科技、泡沫、周期
   - 方法论类：买点、卖点、仓位、杠杆、做T
   - 问答标记（连麦答疑场景）："我""你""问""答"

### 2.2 知识提取（精读阶段）

从文本中提取**可沉淀为知识**的内容，剔除**噪音**：

#### 应该提取的
- **框架性观点**：完整的论述链（前提→推理→结论→操作建议）
- **反直觉洞察**：与主流认知相悖且逻辑自洽的观点
- **量化数据**：具体数字、比例、价格区间、时间节点
- **案例与反例**：过往操作、成功/失败经验
- **情绪与信仰表达**：能折射人格特质和投资哲学的言论

#### 应该忽略的
- 游戏实况细节（"打怪" "出装备" "附魔" 等）
- 个人生活闲聊（吃饭、睡觉、买房子、办过户等）
- 弹幕互动调情（感谢礼物、调侃观众）
- 重复/冗余论述（同一个观点在直播中被反复讲 5 遍以上，只取最佳阐述）
- 明显情绪化吐槽（"我恨死了" "不想活了" 等戏剧性表达，除非能揭示其投资哲学）

### 2.3 结构化输出（写作阶段）

#### Frontmatter（元数据层）

```yaml
---
source_video: BV1xxx       # BVID
source_title: 原标题         # 含日期
upload_time: YYYY-MM-DD HH:MM:SS
duration_estimate: ~N 分钟/小时  # 根据文本量估算
generated_at: YYYY-MM-DD       # 生成日期
has_finance_content: true/false # 判断依据见 1.1
---
```

#### Body（正文层）—— 采用"倒金字塔 + 主题归并"结构

```
# 标题

## 核心立场（1-3 句话）
- UP 主本场的主要论点/立场
- 它与之前言论的延续或变化
- ⭐ 标记重要度（1-3 颗星）

## 金融与交易知识点（按重要性排序）
### 1. 核心知识点标题（⭐ 数 = 重要性）
- 论点本身
- UP 主的论证过程
- 操作含义/推论
- ⚠️ 如果是纠正或推翻自己之前的观点，要特别注明

### 2. 第二个重要知识点
...

## 关于个人成长（非金融，但作者强调）
- 如果原文的"非金融"内容本身有深度（人生哲学、心态修炼），可保留
- 但保持精简，不应喧宾夺主

## 自评/态度（保留人格色彩）
- UP 主当天的情绪状态
- 对自己观点的自信程度
- 对听众的态度变化

## 其他
- 提到的其他标的清单
- 与之前观点的关联
- 值得未来跟踪的"未完成叙事"
```

#### 关键写作原则

1. **一个知识点 = 一个论证链**。不要只写结论，要写"他为什么得出这个结论"
2. **优先记录首次出现的观点**。如果某观点已被写过 5 次，简写或引用上篇
3. **保留人格色彩**。UP 主的口头禅、独特比喻、情绪爆发——这些才有辨识度
4. **不要追求完美表述**。ASR 有错字是正常的，只需提炼核心意思
5. **区分顺势观点与反共识观点**。反共识观点（"科技泡沫要崩"）才是真正有价值的知识沉淀

## 三、常见陷阱与应对

### 陷阱 1：ASR 字间空格导致的"假语义"

```
原文：我 觉 得 市 场 已 经 到 了 顶 部
理解：我觉得市场已经到了顶部（正确）
误读：市场 经 到 了 顶（错误——别被空格切分误导语义单元）
```

**应对**：按"词"而非"字"理解含义——尤其是投资领域固定短语
（"预期差"="预 期 差"、"赚钱效应"、"猪周期"等）

### 陷阱 2：口头禅被误判为知识点

常见的非知识点口头禅：
- "做 T 不是我的兄弟"——虽是反复讲但不构成新知识
- "成本不重要"——已出现 20+ 次，不需要每次记录
- "牧原一定到 120"——目标价表达不算新知识点

**应对**：只有**首次出现 + 有完整论证链**的才记录为新知识点。后续重复见"重复主题"标记。

### 陷阱 3：游戏内容中的金融隐喻

某些 UP 主会在打游戏时用游戏内容隐喻市场（如"MC 里所有的猪都是能繁母猪"）。

**应对**：这类隐喻不构成体系化知识，可作为"观察"记录但不放入正式知识点。除非隐喻本身揭示了新的市场认知框架。

### 陷阱 4：颠倒内容与噪音比例

在 60 分钟直播中，真正有价值的金融内容可能只有 5-10 分钟。剩余都是闲聊/游戏/互动。

**应对策略**：
- 全文读 1 遍 → 标记"有价值段落" → 只对这些段落做精读
- 不要为了"不遗漏"而去读整篇的 MC 游戏攻略

## 四、场景化处理策略

### 场景 A：开盘/盘中直播（"盯盘场"）

特征：UP 主实时看 K 线、情绪化喊单、数据零碎

```
典型产出：
- 当天的市场情绪（而非预测价值）
- 盘中特殊事件（如大单突袭、板块异动）
- UP 主自己在盯盘时的情绪反应（反映持仓状态）
- 盘后总结（如果有的话）
```

**MD 定位**：标记为 `has_finance_content: true` 但知识密度通常中等偏下。
重点记录**盘中事件**和**UP 主对事件的第一反应**，而非长线观点。

### 场景 B：连麦答疑

特征：问答驱动，知识密度高，但问题质量参差不齐

```
典型产出：
- 多个独立主题（每个连麦者一个问题）
- 知识点的"再打磨"——UP 主在回答中把自己的框架更精确地表达一次
- 新手的常见误区（这本身也是反面洞察）
```

**MD 定位**：最宝贵的场景。每个连麦可能产出一个独立知识点。
按问题的主题而非时间顺序组织内容。

### 场景 C：专题讲座（"今天讲讲..."）

特征：UP 主自选主题，系统性输出

```
典型产出：
- 完整的论证链（前提→数据→推理→结论）
- 定量分析（计算器数据）
- 反问/排比的使用（表达核心信仰）
```

**MD 定位**：最高价值。整篇 MD 应该围绕**一个核心论点**展开，所有知识点都为核心论点服务。

### 场景 D：游戏娱乐场

特征：打游戏为主，金融内容偶发

```
典型产出：
- 零星的市场评论
- 游戏与投资的隐喻
- 前后场景的极低一致性
```

**MD 定位**：标记 `has_finance_content: false`，只摘少数金句。不做深度结构。

## 五、跨文档知识管理视角

### 5.1 跟踪"知识演进"

当你处理同一 UP 主的连续多场直播时，需跟踪以下维度：

- **观点一致性**：他是否在一定时间后推翻/修正之前的观点？
- **认知进化**：他对某个话题的理解是否随市场变化而深化？
- **消息验证**：之前的预判（如"SpaceX 7-9 月崩盘"）是否被后续事件验证？
- **持仓变化**：是否在某个时间点调整了持仓（如清仓煤炭、开仓生猪 2705）？

### 5.2 建立"核心观点索引"

在全量 MD 生成后，可提炼一份**核心观点全景图**：

```
当 UP 主在同一话题讲了 N 次之后，应做一次"聚合"：
- 把所有关于"做空黄金"的观点合并为一条完整论述
- 把所有关于"牧原目标价"的推算合并为一条
- 标注每次提及的时间线和认知演变
```

这是更高阶的整合工作，不在单篇 MD 范围内，但应在单篇中为未来聚合打好基础
（如使用一致的标签、引用前置文章的编号等）。

### 5.3 关于 Git 版本管理

每次新增的 MD 和源 txt **应当一起提交**，且 commit message 应包含：
1. 本次新增的 BVID
2. 核心知识点梗概（方便后续检索）
3. 处理深度（A/B/C）

## 六、处理节奏与成本控制

| 文件大小 | 阅读耗时（LLM） | 产出 MD 大小 | 一篇文章的"思考-产出"比例 |
|----------|----------------|--------------|--------------------------|
| ~20 KB | 快读 | 1-3 KB | 1% |
| ~60 KB | 中等 | 5-8 KB | 0.3% |
| ~130 KB | 深度阅读 | 10-15 KB | 0.12% |

对于大文件（>100KB），建议分段处理或只读关键段落，避免上下文膨胀过早耗尽 token 预算。

## 七、质量自检清单

输出 MD 前，请逐项自查：

- [ ] 是否遗漏了本场任何**首次出现的新观点**？
- [ ] 是否混入了**无论证链的噪音**作为知识点？
- [ ] frontmatter 字段是否完整（尤其是 `has_finance_content` 和 `upload_time`）？
- [ ] ⭐ 标记是否反映观点的真实重要度？
- [ ] 对于重复出现的观点，是否写了**简写版本**而非再次长篇展开？
- [ ] 是否保留了 UP 主的人格化语言，还是把它写成了"教科书"？
- [ ] 是否区分了"UP 主的前瞻预判"和"事后归因"？

## 附录 A：本方法论自身的更新方式

本方法论是**开放迭代**的。当遇到以下情况时，应更新本文档：

1. **新的 ASR 格式**：如音频不再是字间空格格式
2. **新的内容类型**：如需要处理财经类访谈、深度研报解读等
3. **新的输出需求**：如用户要求每月生成一份"核心观点汇总"
4. **发现了本流程的 bug**：如某个判断标准在实践中反复出问题

更新原则：**只增加、不删除、注明版本变更**。
